2. Installing the Required Files

If you are installing from the SVN distribution:

  • Please review the instructions at https://svn01.bigworldtech.com/docs/.

  • A number of the client tools, such as CAT (Client Access Tool) and asset_processor, require Python to be installed.

    Python can be downloaded from http://www.python.org. Python 2.5.2 or greater is recommended.

  • SpaceViewer and CAT require wxPython, a GUI toolkit for Python.

    wxPython can be downloaded from http://www.wxpython.org.

    NOTE: Executables require DirectX and VC80 runtimes to be installed. Redistributable installations for both are included in the directx-redist and vc80-redist folders at the root of BigWorld package.

If you are installing using the Setup Wizard (eval package):

  • To install, run the setup executable and follow the prompts.

  • To uninstall, run the uninstaller, either from the Start Menu or from the Add or Remove Programs Utility.

    NOTE: Executables require DirectX and VC80 runtimes. During the setup process, the wizard will prompt about whether you want to install them.

3. Setting Up the Resource Path

This section is only relevant if you are installing from SVN.

The client and tools require the resource search path to be set before they can run. Depending on how you expect to be using them, choose one of these two ways to set up the resource search path:

  • If you plan to run the client and tools with the FantasyDemo's game resources only, edit paths.xml to point to FantasyDemo's and BigWorld's resource directories. See example below: 

    <root>
   <Paths>
      <Path> c:/mf/fantasydemo/res </Path>
      <Path> c:/mf/bigworld/res    </Path>
   </Paths>
</root>

  • If you plan to run the client and tools with your own game resources, in addition to FantasyDemo's game resources, you are better off creating a shortcut to the client and tools pointing to each of the resource paths to be used. The resource search path can be passed to the programs via command line arguments by using the --res switch. See the example below:

    • To run the client with FantasyDemo resources: 

      fantasydemo.exe --res c:/mf/fantasydemo/res;c:/mf/bigworld/res

    • To run the client with YourGame resources: 

      fantasydemo.exe --res c:/mf/yourgame/res;c:/mf/bigworld/res

    • To run WorldEditor with YourGame resources: 

      worldeditor.exe --res c:/mf/yourgame/res;c:/mf/bigworld/res

For details on how to set the resource search path, see Starting the Tools in the Content Tools Reference Guide.

4. Running FantasyDemo

NOTE: For a list of keyboard shortcuts available in FantasyDemo, press F1.

To run FantasyDemo, follow the steps below:

  • Use Windows Explorer to navigate to folder fantasydemo\game.

  • Run the executable fantasydemo.exe.

  • You should be able to connect to our online demo server (by selecting the option 'Connect to Demo Server' in the main menu).

    • You can explore the demo in offline mode by selecting the option 'Explore Space Offline' from the main menu. When offline, however, there will be no entities for you to interact with.

    • If you have difficulties connecting, then please email BigWorld Technical Support at support@bigworldtech.com.

  • The world models day and night cycles, so when you start it, it may be night time, in which case the visibility will be low.

    For the purposes of the demo, we allow you to alter the world time:

    • To move forward through the daytime, hold CAPS LOCK and press [ (LEFT BRACKET).

    • To move backward through the daytime, hold CAPS LOCK and press ] (RIGHT BRACKET).

    You can also hold SHIFT to speed up time.

    Please note that game time is set to run much faster than real-time.

Collaboration

  • Get a colleague to install the demo and log in.

  • Target your colleague, and press NUMPAD DELETE to initiate a handshake.

    The other player can accept a handshake by doing the same, or by clicking while having your avatar targeted.

  • Press ENTER to bring up the chat console.

    Pressing ENTER again without typing anything will close the window.

    Several console commands are available during chat mode (type /help to list them).

5. QuickStart Guide to ModelEditor

To run ModelEditor, follow the steps below:

  • Use Windows Explorer to navigate to folder bigworld\tools\modeleditor.

  • Run the executable modeleditor.exe.

  • Select a model:

    • In the Asset Browser, select the models folder characters/npc/striff.

    • In the list of thumbnails, double-click the file ranger_body.

      The second button on the toolbar performs "Zoom extents". This is helpful if you want to view the whole model, or if the camera is lost.

  • Manipulate the camera:

    • To swivel the camera, move the mouse while pressing the right button.

    • To move the camera, press the WASDQE keys.

  • Click the third button in the toolbar ("Orbit camera") to spin the camera around the model.

  • Play an animation on the model:

    • Select the Animations tab.

    • In the Animations drop-down list, select an entry.

    • Vary the fields Pause Before Loop and Frame Rate.

For more details on ModelEditor, see Content Tools Reference Guide, section ModelEditor.

6. QuickStart Guide to the WorldEditor

To run WorldEditor, follow the steps below:

  • Use Windows Explorer to navigate to folder bigworld\tools\worldeditor.

  • Run the executable worldeditor.exe.

  • Manipulate the camera:

    • To swivel the camera, move the mouse while pressing the right button.

    • To move the camera, press the WASDQE keys.

  • On the toolbar, locate the drop-down list Selection Filter (the last field to the right of the toolbar), and select entry All.

  • Move the cursor over an object until WorldEditor displays its blue bounding box.

  • Left-click the item to select it.

  • While pressing CTRL, move the cursor over other objects, until WorldEditor displays their blue bounding boxes.

    Left-clicking will add them to the list of selected items.

  • Try the gizmos:

    • Position gizmo

      Drag any of the axes in the gizmo to move the selected item in that direction.

      The selected item will move, according to the Alignment specified in the toolbar (set by buttons Free Positioning, Terrain Lock, and Obstacle Lock).

      To drag items along the terrain, select the toolbar button Terrain Lock.

    • Rotation gizmo

      Activate the item's rotation gizmo by pressing SHIFT while it is selected.

      Drag any of the circles of the 3D gizmo to rotate the selected item in that orientation.

    • Scale gizmos

      Activate the item's scale gizmos by pressing ALT while one of them is selected.

      • Regular (non-uniform) scale gizmo

        Drag any axis of the 3D gizmo to resize any one dimension of the selected item.

      • Uniform scale gizmo

        Drag the circle around the selected item inwards or outwards to uniformly resize all dimensions of the item.

  • Test the chunk visualisation modes.

    Press F6 to cycle through the following modes:

    • Chunk borders—Overlays a continuous white line around the border of the chunk (measuring 100m x 100m) over which the mouse is positioned.

    • Pole markers—Divides the chunk over which the mouse is positioned into 25 squares (measuring 4m x 4m each), and overlays pole marker icons on the corner of each one.

    • Wireframe—Divides the chunk over which the mouse is positioned into 25 squares (measuring 4m x 4m each), and overlays the wireframe grid indicator over each one.

    • Normal—Displays the terrain as it will be rendered in the scene. No overlay is applied on the interface.

  • Test the terrain texture brush.

    • Click the Paint terrain textures tool (2).

    • Double click on a texture from the asset locator.

    • Specify the values for the brush's Size and Strength.

    • Left-click the area over which you want to apply the selected texture.

  • Test the terrain drawing.

    • Click the Change Terrain height tool (3).

    • Specify the values for brush's Size and Strength.

    • Set the value of field Explicit Height.

    • Change the terrain's height:

      • Middle-click the terrain to lower it.

      • Left-click the terrain to raise it.

      • Left-click the terrain while pressing LEFT CTRL to set it to a specified height.

For more details on WorldEditor, see Content Tools Reference Guide, section WorldEditor.

7. QuickStart Guide to Running the Server

To run the server (SVN distribution),follow the steps below:

  • Install the server according to the Server Installation Guide.

  • Start BWPanel.

    For details on BWPanel, see Server Operations Guide, section Admin Tools, BWPanel.

  • In BWPanel, click Start the World - Brief.

  • Use the client's "Search for servers in LAN" function to locate and log into your own world.

8. Committing files to a Version Control System

In general, every file included in your package may be checked into a version control system such as CVS, SVN or Perforce. However if the repository you are using uses a lock-before-editing system such (i.e. files are read-only on your hard-disk before use), then there are some files you should leave out. And regardless of whether the repository marks files as read-only by default, there are also some other types of files that are better left uncomitted. To install BigWorld into any version control system, please read the following guide to which files you should commit, and which files you should leave out:

Do not commit these files!

The files below contain user preferences, and are automatically saved to disk on the local user's machine. You don't want these files checked into your repository due to the conflicts that may occur if people accidentally check in these files. Additionally if your repository marks the files as read-only, then the user preferences may not be saved out by the application unless you have unlocked them first - this will affect custom layout of your gui panels, your graphics and video display preferences.

  • bigworld/tools/worldeditor/options.xml

  • bigworld/tools/worldeditor/graphics_preferences.xml

  • bigworld/tools/particleeditor/options.xml

  • bigworld/tools/modeleditor/options.xml

  • fantasydemo/game/preferences.xml

The Asset Browser will also generate additional files for the tools to store the history of assets used in those tools and your favourites. These do not need to be commited into the repository either.

  • bigworld/tools/worldeditor/resources/ual/history.xml

  • bigworld/tools/worldeditor/resources/ual/favourites.xml

  • bigworld/tools/particleeditor/resources/ual/history.xml

  • bigworld/tools/particleeditor/resources/ual/favourites.xml

  • bigworld/tools/modeleditor/resources/ual/history.xml

  • bigworld/tools/modeleditor/resources/ual/favourites.xml

Once you run an application, various log files may be created (such as python.log, or worldeditor.log). These should not be committed either

  • *.log

The next set of files are created when you create a space in WorldEditor. They refer to local changes to a space and are not meant to be checked into the repository.

  • your_game_name/res/spaces/your_space_name/space.localsettings

When you run the game client or the tools, some resources will be created on-disk, as 'processed' or 'compiled' versions of source files. These files are regenerated on demand based on comparing the timestamp of the source with the timestamp of the automatically generated file. If you cannot guarantee that your version control system retains the modified date of files across multiple users' machines, then you should not check in the following types of files :

  • compressed texture map files (.dds)

  • compressed animation files (.anca)

  • generated font bitmaps or processed font files (font/*.dds;*.font)

Commit these files!

Everything else that comes in your package may be committed to your version control system. Additionally, some files that are generated during use may also be committed (these are optional).

  • The following two files must be committed together, or not at all :

  • - your_game_name/res/spaces/your_space_name/space.thumbnail.dds

  • - your_game_name/res/spaces/your_space_name/space.thumbnail.timestamps
