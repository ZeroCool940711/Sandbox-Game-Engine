Detailed Description

The class Base represents an entity residing on a base application. Base entities can be created via the BigWorld.createBase function (and functions with createBase prefix). An base entity instance can also be created from a remote cellApp BigWorld.createEntityOnBaseApp function call.

A base entity can be linked to an entity within one of the active cells, and can be used to create an associated entity on an appropriate cell. The functionality of this class allows you to create and destroy the entity on existing cells, register a timer callback function to be called on the base entity, access contact information for this object, and also access a CellEntityMailBox through which the base entity can communicate with its cell entity (the associated cell entity can move to different cells as a result of movement of the cell entity, or load balancing).

Member Function Documentation

def addTimer( self, initialOffset, repeatOffset=0, userArg=0 ):

The onTimer function must be defined in the base entity's class and accept 2 arguments; the internal id of the timer (useful to remove the timer via the "delTimer" method), and the optional user supplied integer "userArg".

Example: 
# this provides an example of the usage of addTimer
import BigWorld
 
class MyBaseEntity( BigWorld.Base ):
 
    def __init__( self ):
        BigWorld.Base.__init__( self )
 
        # add a timer that is called after 5 seconds, and then every
        # second with custom user data of 9
        self.addTimer( 5, 1, 9 )
 
        # add a timer that is called once after a second, with custom
        # user data defaulting to 0
        self.addTimer( 1 )
 
    # timers added on a Base have "onTimer" called as their timer function
    def onTimer( self, id, userArg ):
        print "MyBaseEntity.onTimer called: id %i, userArg: %i" % ( id, userArg )
        # if this is a repeating timer then call:
        #     self.delTimer( id )
        # to remove the timer when it is no longer needed

Parameters:
initialOffset initialOffset is a float and specifies the interval (in seconds) between registration of the timer, and the time of first call.
repeatOffset repeatOffset is an optional float and specifies the interval (in seconds) between repeated executions after the first execution. The delTimer method must be used to remove the timer or this will repeat continuously. Values less than 0 are ignored.
userArg userArg is an optional integer which specifies an optional user supplied value to be passed to the "onTimer" method of the base entity.

Returns:
Integer. This function returns the internal id of the timer. This can be used with delTimer method to remove the timer.

def createCellEntity( self, cellEntityMB ):

The information used to create the cell entity is stored in the cellData property of this entity. This property is a dictionary corresponding to the values in the entity's .def file plus a "position", "direction" and "spaceID" entries for the entity's position and (roll, pitch, yaw) direction.

If cellEntityMB is not passed in, the "spaceID" entry in cellData is used to indicate which space to create the cell entity in.

Parameters:
cellEntityMB an optional CellEntityMailBox argument which is used to indicate which space to create the cell entity in. Ideally, the two entities are near so that it is likely that the correct cell will be found immediately.

def createInDefaultSpace( self ):

The information used to create the cell entity are stored in the cellData property of this entity. This property is a dictionary corresponding to the values in the entity's .def file plus a "position", "direction" and "spaceID" entries ("spaceID" entry is not referenced in this method) for the entity's position and (roll, pitch, yaw) direction.

This method is only available if useDefaultSpace is set to true in the server's configuration file.

def destroy( self, deleteFromDB, writeToDB ):

It may be convenient to call self.destroy in the onLoseCell callback. This causes the base entity to be destroyed whenever the cell entity is destroyed.

This method has two keyword argument:

Parameters:
deleteFromDB If True, the entry in the database associated with this entity is deleted. This defaults to False.
writeToDB If True, the properties associated with this entity are written to the database. This only occurs if this entity was read from the database or has previously been written with Base.writeToDB. This defaults to True but is ignored if deleteFromDB is True.

def teleport( self, baseEntityMB ):

On arrival in the new space, Entity.onTeleportSuccess is called. This can be used to move the entity to the appropriate position within that space.

Parameters:
baseEntityMB The base mailbox of the entity that this entity should be moved to. The cell entity associated with this argument will be passed into Entity.onTeleportSuccess on success.

def writeToDB( self, callback, shouldAutoLoad, shouldWriteToPrimary ):

Entities can also be marked as auto-loaded, so that subsequent server start-ups will recreate this entity automatically. This auto-load data can be cleared from the database using the clear_auto_load tool. See the Server Operations Guide -> MySQL Support -> The clear_auto_load tool for more details.

Parameters:
callback This optional argument is a callable object that will be called when the response from the database is received. It takes two arguments. The first is a boolean indicating success or failure and the second is the base entity.
shouldAutoLoad This optional argument specifies whether this entity should be loaded from the database when the server starts up.
shouldWriteToPrimary This optional argument specifies whether this write should be sent to the primary database instead of the secondary database (defaults to False).
