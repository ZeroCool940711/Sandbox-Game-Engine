GUI Module Reference

Member Function Documentation

def AlphaShader( ):

This function creates a new AlphaGUIShader. An AlphaGUIShader is used to control the alpha on a GUI component and its children.

Returns:
a newly created AlphaGUIShader.

def Attachment( ):

Creates and returns a new GUIAttachment, which is used to display gui elements in the 3D scene.

Returns:
The new GUIAttachment object

def BoundingBox( textureName ):

Parameters:
textureName The full resource path of the texture to be used.

Returns:
The newly created BoundingBoxGUIComponent

def ClipShader( ):

This function creates a new ClipGUIShader. This is used to clip the specified portion off one side of the component.

Returns:
a new ClipGUIShader.

def ColourShader( ):

This function creates a new ColourGUIShader. This is used to apply colours to gui components.

def Console( ):

Creates and returns a new ConsoleGUIComponent, which displays an editable console on the screen.

Returns:
The new ConsoleGUIComponent.

def Flash( movie ):

This function creates a new FlashGUIComponent which allows a flash movie to be inserted into the GUI hierarchy. If this component has an attached script, then the script must take responsibility for passing mouse and key events into the PyMovieView object.

Parameters:
movie A PyMovieView object containing the movie to associate with the component.

Returns:
the newly created component.

def FlashText( text ):

This function creates a new FlashTextGUIComponent. This component renders a line of text.

Parameters:
text A string containing the line of text to be rendered. This can be changed once the component is created.

Returns:
the newly created component.

def Frame( backgroundTextureName, cornerTextureName, edgeTextureName, tiledWidth, tiledHeight ):

This function creates a new FrameGUIComponent. This is basically a tiled simpleGUIComponent with a frame around it. The frame is specified by and edge texture and a corner texture. The size of the frame is specified by the tiledWidth and tiledHeight, which are also used to specify the tiling of the background.

Parameters:
backgroundTextureName The filename of the background texture. This needs to be a valid texture file.
cornerTextureName The filename of the corner texture. This needs to be a valid texture file.
edgeTextureName The filename of the edge texture. This needs to be a valid texture file.
tiledWidth The width to make the frame, and to tile the background at.
tiledHeight The height to make the frame, and to tile the background at.

def Frame2( ):

This function creates a new FrameGUIComponent2. This is a resizable frame created from a single texture and a single draw call. The frame requires specific artwork, please see the content creation guide for details.

def Gobo( textureName ):

This function creates a new Gobo component. The GoboComponent acts like other gui components, but blends the gui component texture with another specified PyTextureProvider, using the alpha channel of the component's texture.

If you associate a render target with the gobo component, you can also freeze and unfreeze the secondary PyTextureProvider.

A recommended use of the gobo component is to create a post-processing effect, and use the output of that effect as the second texture stage in the gobo component.

This is for example perfect for binoculars and sniper scopes.

For example, in the following code: 

comp = GUI.Gobo( "gui/maps/gobo_binoculars.dds" )
comp.materialFX="SOLID"
comp.secondTexture=myBlurredRenderTarget.texture
GUI.addRoot( comp )


This example will display a binocular gobo, and where the alpha channel is relatively opaque in the binocular texture map, a blurred version of the scene is drawn.

Parameters:
textureName The filename of the gobo texture. The alpha channel determines how much of the blurred screen should be blended in.

def addRoot( component ):

This function adds a GUI component to the gui root. GUI elements exist in a treelike structure, with every element having a parent. The top level parent is the root of the whole tree. This root exists internal to the GUI module. In order for a gui component to be processed, for both rendering and event handling, it, or one of its parents, must be added to the root.

Parameters:
component the gui component to add to the root

def delRoot( component ):

This function removes a Gui component from the root gui's list of children.

Parameters:
component the component to remove

def draw( ):

This method causes the GUI and all components which have been added as roots to be redrawn.

It is called automatically by the system each tick. It would not normally be necessary to call this function.

def handleAxisEvent( event ):

This function gets the GUI to handle the given axis event. It hands the event to each of its child components which have the focus attribute set, until one of them returns non zero, at which point it returns non-zero. Otherwise, it returns zero.

This needs to be called from the personality script, otherwise the GUI will not process axis events.

Parameters:
event A PyAxisEvent object.

Returns:
non-zero means the event was handled, zero means it wasn't.

def handleAxisEvent( event ):

If present, this method is called whenever a joystick axis event occurs.

* This needs to be called from the personality script, otherwise the GUI will * not process axis events.

Parameters:
event A PyAxisEvent object.

Returns:
This function should return False if the axis event was not handled * and True if the axis event was handled.

def load( section ):

This module function loads a gui component from a data section. The data section can either be specified as a string resourceID (filename), or as a PyDataSection. Normally, the data section would have been created previously by calling the save method on the gui component.

Parameters:
section either a resourceId or a PyDataSection

def mcursor( ):

This function returns the MouseCursor object.

Use it, in conjunction with BigWorld.setCursor function to activate the mouse cursor. The mouse cursor object should be activated whenever is desirable to have the user interacting with GUI components by using the mouse. No mouse events will be propagated to the components if the mouse cursor is not active. To activate the mouse cursor, use: 

	BigWorld.setCursor( GUI.mcursor() )


Also, use it to access and modify all mouse cursor properties, like position, visible, shape and active: 

	mouseStatus = GUI.mcursor().active
	mousePosition = GUI.mcursor().position
	GUI.mcursor().visible = True
	GUI.mcursor().shape = 'arrow'

def printRoots( ):

Prints the GUI structure (as info msgs) this is useful for debugging GUI structures.

def reSort( ):

This function resorts the gui components at the root of the GUI tree. This function sorts according to the third (depth) component of the children's position attributes. Changing this depth value in a child doesn't automatically reorder the children, so an explicit call to this function is required.

Returns:
Read-Only list of GUI components.

def roots( ):

This function returns a list of the current root components. It is not editable. Instead, use the GUI.addRoot and GUI.delRoot to add and remove components from the list.

Returns:
Read-Only list of GUI components.

def screenResolution( ):

This function simply returns the current (width, height) in pixels of the screen. If a resolution override has been set, then this returns the override. Use BigWorld.screenSize if you need the physical resolution regardless of whether the GUI resolution has been overridden.

Returns:
A 2-tuple representing the width and height of the screen, in pixels.

def setDragDistance( distance ):

Sets the minimum distance the mouse pointer has to travel after the left mouse button has been pressed before the movement is considered a drag.

Once the movement is classified as dragging, a click event will no longer be generated if the mouse is released over the dragged component.

Parameters:
distance the minimum drag distance (float, in clip space). The default value is 0.002.

def setResolutionOverride( resolution ):

This method overrides the resolution that the GUI system uses for calculations. This allows components setup in pixel space to be automatically scaled. For example, if the resolution override is set to 1024x768, then the GUI will take up the same area of the screen no matter what the actual screen resolution is.

Parameters:
resolution A Vector2 of the new override. If it is a zero vector then the override is disabled.

def update( dTime ):

This function causes the GUI to be updated as if the specified amount of time had passed. This updates all of the components and shaders which have been added to the GUI root.

The system automatically updates the gui each tick with dTime on each update being the actual time since the last update. This call would not normally need to be used.

Parameters:
dTime The amount of time assumed to have passed since the last update.
