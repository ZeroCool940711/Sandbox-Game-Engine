The Python Console module provides the ability to access Python functionality (and even exposed C++ methods) from the WebConsole without needing to connect to a linux machine and use developer / systems operator oriented tools.

1. Python console

The Python Console page provides a simple web interface to the Python telnet port of BaseApps and CellApps. Using the Python console, users can interactively call script commands on a server process, or inspect the state of the server.

Previously, developers would have to use `control_cluster.py pyconsole`, or run telnet from the command line to access this functionality.

1.1. Usage

Opening the Python Console page presents the user with a list of active server components owned by the current user that have an exposed Python console.

Each process has a Connect link which links to a terminal like page that is connected to the associated server component. The console page can also be accessed via the Cluster Control module's Manage Servers page, and in the process Action drop-down list box selecting Console entry.

The Python console page is designed to be similar to an interactive command line python console that most python developers should have encountered.
